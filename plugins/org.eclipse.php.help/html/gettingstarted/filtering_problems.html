Filtering Problems

Filter

You can filter problems to limit the view to a subset of only those problems that are of immediate interest. This feature is generally used when the workspaces has too many problems to view them all effectively.

Note:  All problems recognized by the PDT can be enabled/disabled. Filtering enables you to view a user-defined subset.

  1. Click on the Filter icon on the Problems View tab. The Problems Filter will open.

Figure 2 - Problem Filters

Problem Sets

Select one of the following problem sets:

  • Show problems for any resource.

  • Show problems for any resource in the same project as the selected resource.

  • Show problems for the selected resource only.

  • Show problems for the selected resource and its children.

  • Show resource for a specific working set

  • Filter according to description

Severity

Filter according to the problem severity:
  • Error
  • Warning
  • Info.

Problem Types

The Filter displays / hides the following types of problems:

  • APT Build Marker (XML related problems, not defined tasks / bad format)
  • APT Configuration Marker
  • EAR Validator  (J2ee)
  • EJB Validator Message  (J2ee)
  • EMF problems
  • Faceted Project Problem
  • Faceted Project Problem (Java)
  • Faceted Project Problem (Mis*)
  • Java Build Path Problems (Java/ PHP include path problems)
  • Java problems
  • PDE problems (Plugin Development Environment)
  • Problem
  • Validation Message: Web files syntax problems which found using a validator are registered under this problem item. PHP is one of them.
  • XML schema problems (Document Type Definition) / XML Scheme Definitions)
